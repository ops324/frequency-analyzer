# Frequency Analyzer — 仕様書

リアルタイム音響解析ツール「Frequency Analyzer（Observatory）」の仕様書。
マイク入力から周波数・音名・チューニング（セント偏差）を解析し、シネマティックな3Dビジュアライザーで可視化する単一HTMLアプリ。

- **最終更新:** 2026-05-24（ディスプレイフォントを EB Garamond に変更）
- **形式:** 単一HTMLファイル（ビルド不要）
- **公開URL:** https://frequency-analyzer.vercel.app （Vercel・GitHub連携で自動デプロイ）

---

## 1. 概要

| 項目 | 内容 |
|------|------|
| 目的 | マイク音声をリアルタイム解析し、周波数・音量・音名・チューニングを表示・可視化する |
| 形態 | 単一HTML（`index.html`）。サーバー不要、静的ホスティングで動作 |
| デザイン方針 | シネマティック天文台（Cinematic Observatory）— 漆黒の宇宙的背景、ブルーム発光、フィルムグレイン、暖色×寒色のデュオトーン |
| 対象ブラウザ | Chrome / Edge / Safari / Firefox の最新版（WebGL2 + Web Audio API対応） |
| 動作要件 | マイク使用には**セキュアコンテキスト**（`https://` または `localhost`）が必須 |

## 2. 機能仕様

### 2.1 計測・表示
- **Frequency（周波数）**: 検出基本周波数を `Hz`（小数1桁）で表示。値は補間（lerp）で滑らかに更新。
- **Volume（音量）**: RMSから算出した `dB` を整数表示。
- **Note（音名）**: 検出周波数に最も近い音名＋オクターブ（例: `A4`）を表示。
- **Tuner（チューナー）**: 最近接音からの**セント偏差（±50）**を算出。
  - SVG円弧ゲージ＋イージング付き針で偏差を可視化。
  - `|cents| < 5` で「In Tune」判定 → 針・円弧・音名がミントグリーンに発光（リワード演出）。
- **Spectrum（スペクトラム）**: 対数周波数軸（40Hz–18kHz）の2Dバー表示。寒色→暖色グラデーション。

### 2.2 ビジュアライザー（Three.js）
- 二重リングの放射状リアクティブバー（周波数ビンに連動、暖色↔寒色マッピング）。
- 中央コア（正二十面体＋ワイヤーフレーム）が音量（RMS）に連動して脈動。
- 波形リング（LineLoop）が時間波形を描画。
- スターフィールド（点群）＋フォグによる奥行き、フロアグリッド。
- **UnrealBloomPass** による実ブルーム発光。**閾値（threshold）を高め**に設定し、発光を明部（明るいバー・コア）だけに限定 → 暗部は締まったまま白飛びしない。強度は音量で微増。
- シネマティックなカメラドリー＋マウスパララックス。
- 無音時も「呼吸する」アイドルアニメーションを継続。

### 2.3 UI / インタラクション
- **イントロオーバーレイ**: 初回は「Press to listen」。クリックでマイク許可を要求（ユーザージェスチャ要件を満たす）。段階的フェードイン演出。
- **トランスポート**: コンソール内の Listen / Stop トグル＋ステータスドット。
- **テーマ切替**: ダーク（Night / 既定）↔ ライト（Day / 暖色のスタジオ調）。`applyTheme()` が3Dシーン全体を一括最適化 — クリアカラー・フォグ・トーンマッピング露出・ブルーム閾値/強度・各ライト強度・バーの色と発光量・コア/波形/星/グリッドの色を**モード別に切替**。
  - ライトでは白飛びを避けるため、ブルームをほぼ無効化＋バーを暗め・高彩度・低発光にして明背景でも視認可能にする「昼スタジオ」処理。
- **HUD**: 四隅にブランド・サンプルレート・FFTサイズ・ステータスをモノスペースで表示。

### 2.4 状態とエラー処理
| 状態 | 挙動 |
|------|------|
| Ready | イントロ表示、ビジュアライザーはアイドル動作 |
| Listening | マイク解析中。各値・ゲージ・スペクトラム・3Dが反応 |
| Stopped | 解析停止、表示は「—」にリセット、スペクトラムクリア |
| マイク拒否 | イントロ内に「Microphone access denied …」を表示（クラッシュしない） |

## 3. 技術仕様

### 3.1 技術スタック
- HTML / CSS / Vanilla JS（ESモジュール、ビルド不要）
- **Three.js 0.160.0**（`<script type="importmap">` + ESM、CDN: jsDelivr）
  - 後処理: `EffectComposer` / `RenderPass` / `UnrealBloomPass` / `OutputPass`
- **Web Audio API**（`AnalyserNode`, `getUserMedia`）
- フォント: `EB Garamond`（ディスプレイ）/ `Geist Mono`（数値・データ、tabular figures）— Google Fonts

### 3.2 ピッチ検出アルゴリズム
- **NSDF（McLeod Pitch Method）** ベースの自己相関。
  - `fftSize = 4096`、`smoothingTimeConstant = 0.8`。
  - 探索レンジ: 約 60Hz–1400Hz（`maxLag = sr/60`, `minLag = sr/1400`）。
  - RMS閾値 `0.003` 未満は無音として無視。
  - 放物線補間でピーク位置を精緻化。
- セント計算: `cents = (12·log2(f/440)+69 − round(...)) × 100`、±50にクランプ。

### 3.3 パフォーマンス / アクセシビリティ
- `setPixelRatio(min(devicePixelRatio, 2))` で解像度を抑制。
- `composer` / キャンバスはリサイズ時に再構成。
- 停止時に `MediaStream` / `AudioContext` を解放。
- `prefers-reduced-motion` でグレイン・パララックス・各種アニメーションを無効化。
- セマンティックなボタン、`aria-label`、`:focus-visible` リング対応。
- **コントラスト（WCAG）**: 両テーマで AA 準拠を確認。主要数値・音名 ≈17:1、cents 6.8–8.1:1、微小ラベル（ラベル/単位/ステータス）はダーク5.0:1・ライト5.4:1。半透明パネルは不透明度・縁・影を強化し3D背景から確実に分離。

## 4. デザインシステム

### 4.1 カラーパレット
| トークン | ダーク | ライト | 用途 |
|---------|--------|--------|------|
| 背景 (--bg) | `#04060b` | `#e4ddcd` | ページ／ステージ |
| 3Dクリア (--clear) | `#04060b` | `#d8cfba` | WebGLクリアカラー（ライトはやや沈めてバーを視認可能に） |
| テキスト | `#f1f3f8` | `#1a1712` | 主要テキスト |
| 薄文字 (--text-faint) | `#7b8499` | `#6f6654` | ラベル/単位（AA準拠まで引き上げ） |
| Amber（主アクセント） | `#f2c17d` | `#a4630f` | 周波数・主要発光 |
| Cool（寒色） | `#7cc4e2` | `#256f93` | 奥行き・寒色側マッピング |
| Mint（成功） | `#5fe3a4` | `#0f8a55` | **In Tune時のみ**使用 |

### 4.2 タイポグラフィ
- ディスプレイ: `EB Garamond`（タイトル、音名）— 端正で上品なクラシックセリフ体
- データ／数値: `Geist Mono`（フォールバック `JetBrains Mono`、等幅・tabular-nums）
- マイクロラベル: モノスペース、大文字、ワイドトラッキング

### 4.3 モーション
- ページロード: 段階的リビール（`animation-delay` によるスタガー）。
- 数値: lerp による滑らかな追従（ジッター抑制）。
- グレイン: ステップアニメーション。針: イージングで目標値へ収束。

## 5. ファイル構成

```
周波数測定器/
├── index.html            # 本体（マークアップ・CSS・描画/音声JS）。純粋関数は src/ から import
├── src/
│   └── pitch.js          # ピッチ/音楽理論の純粋関数（freqToNote / centsFromNearest / detectPitch）
├── tests/
│   └── pitch.test.mjs    # node:test によるユニットテスト
├── package.json          # npm test（node --test）。ビルド依存なし
├── 仕様書.md             # 本ドキュメント
├── .github/workflows/
│   └── ci.yml            # GitHub Actions（push / PR でテスト実行）
└── .claude/              # 開発用ヘルパー（プレビューサーバー設定）
    ├── launch.json
    └── static-server.cjs
```

> **テスト**: `npm test`（依存ゼロ、Node 標準 `node:test`）。`detectPitch` は合成正弦波からの周波数復元と無音判定を検証。CI（`.github/workflows/ci.yml`）が `main` への push と PR で自動実行。

## 6. 起動・デプロイ

### 6.1 ローカル起動（マイク利用にはセキュアコンテキストが必須）
```bash
cd 周波数測定器
python3 -m http.server 8000
# → http://localhost:8000/ を開く
```

### 6.2 デプロイ（Vercel）
- リポジトリ `ops324/frequency-analyzer`（公開）に Vercel が連携し、`main` への push で**自動デプロイ**。
- 静的ファイル配信（ビルド設定不要）。公開URL: **https://frequency-analyzer.vercel.app**（HTTPS = セキュアコンテキストのためマイク利用可）。
- 更新手順: `git push origin main` → Vercel が Production ビルド → 公開URLに反映（通常1〜2分）。
